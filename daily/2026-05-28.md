# Day 7: Chain-aware Context 实践

## 📋 实践目标

给一笔 Uniswap 交易做上下文包，验证 Chain-aware Context 的设计。

## 🎯 选择的交易

**交易哈希**: `0xa40162ed6da4c2fdc7645f381600a185c0ed7efe45e98ffae54449148ff01f81`


**为什么选这笔交易**：
- Uniswap V3 Swap 交易
- 包含 token transfers（USDC → WETH）
- 包含多个 logs（Swap, Transfer 事件）
- 有合约调用（SwapRouter）
- 数据结构清晰，适合学习

**Etherscan 链接**: 
- Mainnet: https://etherscan.io/tx/0xa40162ed6da4c2fdc7645f381600a185c0ed7efe45e98ffae54449148ff01f81
- （注意：这是示例交易哈希，你需要替换为真实的）

## 📝 实践步骤

### 阶段 1：手动收集数据（1 小时）

**步骤 1：找一笔真实的 Uniswap 交易**
1. 去 https://etherscan.io/
2. 搜索 Uniswap V3: SwapRouter 合约
3. 找到最近的一笔 Swap 交易
4. 记录交易哈希

**步骤 2：收集交易数据**
从 Etherscan 手动收集：
- [ ] Chain ID（1 = Ethereum Mainnet）
- [ ] Block Number
- [ ] From（发起者地址）
- [ ] To（合约地址）
- [ ] Method（函数名，比如 `exactInputSingle`）
- [ ] Value（ETH 数量）
- [ ] Token Transfers（哪些代币转移了）
- [ ] Logs（事件日志）

**步骤 3：找合约 ABI**
- [ ] 在 Etherscan 找到合约的 Verified Source
- [ ] 复制相关函数的 ABI
- [ ] 理解函数参数的含义

**步骤 4：设计 Context 文档**
按照 Day 4 的 6 层结构，设计 Context：

```markdown
# Transaction Context: [交易哈希]

## 1. 指令层（System Rules）
- Agent 的角色：交易分析助手
- 输出格式：结构化的交易分析报告
- 安全原则：只分析链上数据，不做投资建议

## 2. 任务层（Task Context）
- 用户意图：理解这笔交易做了什么
- 分析目标：解释交易的业务逻辑

## 3. 事实层（On-chain Facts）⛓️
**标记说明**：⛓️ = 链上可验证事实

- ⛓️ Chain ID: 1 (Ethereum Mainnet)
- ⛓️ Block Number: [区块号]
- ⛓️ Transaction Hash: [交易哈希]
  - 🔗 Explorer: https://etherscan.io/tx/[哈希]
- ⛓️ From: [地址]
- ⛓️ To: [合约地址]
- ⛓️ Method: [函数名]
- ⛓️ Value: [ETH 数量]
- ⛓️ Token Transfers:
  - Transfer 1: [数量] [代币] from [地址] to [地址]
  - Transfer 2: ...
- ⛓️ Logs:
  - Event 1: [事件名]([参数])
  - Event 2: ...

## 4. 知识层（Knowledge Base）
**标记说明**：📚 = 外部知识

- 📚 Uniswap V3 SwapRouter: 去中心化交易所的路由合约
- 📚 USDC: 稳定币，1 USDC ≈ 1 USD
- 📚 WETH: Wrapped ETH，ERC20 版本的 ETH

## 5. 解释层（Interpretation）💭
**标记说明**：💭 = 基于事实的解释

- 💭 这笔交易是一个 Uniswap Swap
- 💭 用户用 [数量] USDC 兑换了 [数量] WETH
- 💭 交易路径：USDC → Pool → WETH
- 💭 Gas 费用：[数量] ETH

## 6. 来源追溯（Source Tracing）
- 所有 ⛓️ 标记的数据来源：Ethereum Mainnet RPC
- 所有 📚 标记的知识来源：Uniswap 官方文档
- 所有 💭 标记的解释基于：⛓️ 事实层数据
```

## 🎯 核心验证点

### 1. 标记系统是否清晰？
- ⛓️ 链上事实
- 📚 外部知识
- 💭 基于事实的解释

### 2. 来源追溯是否完整？
- 每个关键结论都有来源
- 每个链上数据都有 explorer 链接

### 3. Context 分层是否合理？
- 事实层只包含可验证数据
- 解释层明确标注为解释
- 知识层提供必要背景

### 4. 实时性问题如何处理？
- 交易数据是历史数据（不会变）
- 但如果是查询余额、价格等实时数据，如何标注时间戳？
