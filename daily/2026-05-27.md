**核心概念**：
- MCP Server：提供工具和数据源的服务端
- MCP Client：使用 MCP Server 的客户端（AI Agent）
- Tools：MCP Server 暴露的函数
- Resources：MCP Server 提供的数据源
- Prompts：预定义的提示词模板

**MCP 的价值**：
- 标准化：统一的接口规范
- 安全性：明确的权限边界
- 可组合性：多个 MCP Server 可以组合使用

## MCP最小实践提示词：

我要做一个只读 MCP server，它只暴露两个工具：

search_docs(query)：搜索本地项目文档。
get_file(path)：读取白名单目录里的文件。
要求：

不能读取白名单外路径。
返回结果必须带来源路径。
工具调用要写日志。
错误要明确返回，而不是静默失败。
完成后，再设计一个“写入工具”的权限升级方案：什么时候需要用户确认，如何撤销授权，如何审计每次调用。

我的思路：
1、在experiments/mcp-practice里创建项目：
2、使用modelcontextprotocol/python-sdk实现MCP server和client

输入：query
输出：搜索结果，包含文档路径
实现：使用python的os模块搜索项目目录下的所有文件，返回包含query的文件路径

输入：path
