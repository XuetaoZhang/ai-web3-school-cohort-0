## 💻 Practice & Experiments / 实践与实验

### RAG System / RAG 系统

**What I built / 构建内容**:
- 创建了基于 DeepSeek API 的简化版 RAG 系统
- 知识库：7 个 AI × Web3 主题文档（LLM、Prompt、Agent、RAG、Web3 Agent、Context、Embedding）
- 检索方式：简单的关键词匹配（避免复杂依赖）
- 项目结构：
  - `knowledge_base.py` - 知识库定义
  - `rag_simple.py` - 简化版 RAG 系统（推荐）
  - `vector_store.py` - 向量数据库（需要 ChromaDB）
  - `rag_system.py` - 完整版（需要 ChromaDB）
  - `QUICKSTART.md` - 快速开始指南

**Challenges faced / 遇到的挑战**:
1. **依赖冲突**: sentence-transformers 与 torch 版本冲突
2. **ChromaDB 安装失败**: pandas 构建失败，依赖链太复杂
3. **国内 API 访问**: 原指南使用 Claude API，国内无法直接访问

**Solutions found / 找到的解决方案**:
1. **跳过 sentence-transformers**: 使用 ChromaDB 默认 embedding 或简单关键词匹配
2. **创建简化版**: `rag_simple.py` 不依赖向量数据库，使用关键词匹配
3. **使用 DeepSeek API**: 创建 DeepSeek 版本实践指南，API 格式兼容 OpenAI
4. **只安装必要依赖**: 只安装 `openai` 库，避免复杂依赖

## 💡 Key Insights / 关键洞察

1. **RAG 的核心价值**：通过检索外部知识，LLM 可以回答训练数据之外的问题，减少幻觉，提供可追溯的信息来源
2. **技术选型的灵活性**：遇到依赖问题时，可以用简化方案（关键词匹配）替代复杂方案（向量数据库），先理解核心概念
3. **国内 API 选择**：DeepSeek API 在国内可直接访问，API 格式兼容 OpenAI，是 Claude 的良好替代
4. **实践优先**：不要被工具和依赖问题阻塞，先用最简单的方式实现核心功能，理解原理后再优化
