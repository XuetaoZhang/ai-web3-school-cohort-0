# Frameworks 章节核心内容

## 什么是 AI Frameworks？

AI Frameworks 是构建 AI 应用的工具库，提供：
- **抽象层**：隐藏底层 API 调用细节
- **组件库**：预构建的常用组件（Memory、Tools、Chains）
- **最佳实践**：经过验证的架构模式

## 主流 Frameworks

### 1. LangChain
- **定位**：最流行的 AI 应用开发框架
- **核心概念**：
  - Chains：组合多个步骤
  - Agents：自主决策和工具调用
  - Memory：对话历史管理
  - Tools：外部工具集成
- **适用场景**：复杂的多步骤 AI 应用

### 2. LlamaIndex
- **定位**：专注于数据索引和检索
- **核心概念**：
  - Index：数据索引（向量、树、图）
  - Query Engine：查询引擎
  - Retriever：检索器
- **适用场景**：RAG 系统、知识库问答

### 3. AutoGPT / BabyAGI
- **定位**：自主 Agent 框架
- **核心概念**：
  - Task Decomposition：任务分解
  - Self-Reflection：自我反思
  - Memory：长期记忆
- **适用场景**：需要自主规划的复杂任务

## Frameworks 的价值

### 优势
1. **快速开发**：不用从零开始
2. **最佳实践**：避免常见陷阱
3. **生态系统**：丰富的插件和工具

### 劣势
1. **抽象成本**：隐藏了底层细节，难以调试
2. **过度设计**：简单任务可能不需要 Framework
3. **版本迭代快**：API 变化频繁

## 何时使用 Frameworks？

### 适合使用
- 复杂的多步骤应用
- 需要 Memory 管理
- 需要多种工具集成
- 团队协作开发

### 不适合使用
- 简单的单次 API 调用
- 需要极致性能优化
- 需要完全控制底层逻辑
- 原型验证阶段

## 最小实践建议

1. **先理解原理**：不要直接用 Framework，先手写一遍
2. **按需引入**：不要一开始就用全套 Framework
3. **保持简单**：能用简单方案就不用复杂方案
4. **关注底层**：理解 Framework 做了什么

## Web3 场景的特殊考虑

1. **链上数据集成**：Framework 是否支持 Web3 数据源？
2. **交易签名**：如何安全地集成钱包？
3. **实时性**：链上数据变化快，缓存策略如何设计？
4. **Gas 优化**：如何减少不必要的链上查询？
