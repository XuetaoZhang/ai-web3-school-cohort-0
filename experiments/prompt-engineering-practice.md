#### Step 1: 基础 Prompt 对比（15分钟）

**任务**: 对比不同 Prompt 的效果

**实验 1.1: 模糊 vs 清晰**

模糊：这两个对比效果区别很大，模糊的提示词根本就只能让LLM自己输出可能训练数据里最多的数据，此处输出的是一个基本的改查合约函数；
清晰：清晰的提示词则能够引导LLM输出符合要求的代码，此处输出的是一个符合要求的ERC20代币合约。

**实验 1.2: 无示例 vs Few-shot**

Zero-shot：给定一个交易哈希，要求LLM根据训练数据中的分类进行分类。
Few-shot：给定一个交易哈希，要求LLM根据训练数据中的示例进行分类。
对比：这两个对比效果区别很大，Zero-shot的LLM只能根据训练数据中的分类进行分类，而Few-shot的LLM则能够根据训练数据中的示例进行分类。

**实验 2.1: 添加角色**

明确了Solidity 安全审计专家角色：
你是一个专业的Solidity安全审计专家，负责审核Solidity合约的安全性。
你根据Solidity合约的代码，检查合约是否存在安全问题。
但是现在看来这个专家角色的作用不是很大，可能因为没有对比；


**实验 2.2: 添加约束条件**

直接设定了约束条件：
- 代码不超过 30 行
- 包含使用示例

输出的内容明显比无约束准确，没有很多的发散

**实验 3.1: 结构化输出**

约束以 JSON 格式输出，LLM直接输出符合要求的 JSON 字符串，没有多余字段和格式，适合需要解析的场景；

**实验 3.2: Markdown 表格输出**

约束以md格式输出，LLM直接输出符合要求的md表格，没有多余字段和格式，适合需要解析的场景；
