## 🔍 检索方式详解

### 1. 关键词匹配 (`rag_simple.py`)

**原理**：
- 统计查询词在文档中出现的次数
- 标题匹配权重更高
- 简单粗暴，无向量化

**示例**：
```python
query = "AI Agent"
# 在文档中查找 "AI Agent" 出现次数
# 标题包含 "AI Agent" 的文档得分更高
```

**优点**：
- ✅ 无依赖，极简
- ✅ 运行速度快
- ✅ 适合学习 RAG 概念

**缺点**：
- ❌ 无法理解语义
- ❌ 无法处理同义词
- ❌ 查询词必须完全匹配

**适用场景**：
- 快速理解 RAG 概念
- 演示 RAG 工作流程

---

### 2. TF-IDF 向量化 (`rag_vector.py`) ⭐ 推荐学习

**原理**：
- **TF (Term Frequency)**: 词频，词在文档中出现的频率
- **IDF (Inverse Document Frequency)**: 逆文档频率，词的区分度
- **向量化**: 将文档转换为高维向量
- **余弦相似度**: 计算查询向量和文档向量的夹角

**完整流程**：

```
1. 文档向量化
   文档 1: "AI Agent 是智能系统" → [0.2, 0.5, 0.1, ...]
   文档 2: "RAG 是检索增强生成" → [0.1, 0.3, 0.6, ...]

2. 查询向量化
   查询: "什么是 Agent" → [0.3, 0.4, 0.0, ...]

3. 计算余弦相似度
   cos(查询, 文档1) = 0.85
   cos(查询, 文档2) = 0.32

4. 排序
   文档1 (0.85) > 文档2 (0.32)

5. 返回最相关文档
```

**数学原理**：

```
TF-IDF(词, 文档) = TF(词, 文档) × IDF(词)

TF(词, 文档) = 词在文档中出现次数 / 文档总词数

IDF(词) = log(文档总数 / 包含该词的文档数)

余弦相似度 = (向量A · 向量B) / (||向量A|| × ||向量B||)
```

**优点**：
- ✅ 真正的向量检索
- ✅ 考虑词的重要性（IDF）
- ✅ 可以处理部分语义
- ✅ 依赖简单（只需 scikit-learn）
- ✅ 运行速度快

**缺点**：
- ⚠️ 语义理解能力有限
- ⚠️ 无法处理同义词（"汽车" vs "车辆"）
- ⚠️ 中文需要分词才能更好工作

**适用场景**：
- ✅ **学习向量检索流程**（强烈推荐）
- ✅ 理解 embedding、相似度、排序
- ✅ 小规模知识库
- ✅ 对语义理解要求不高的场景

---

### 3. 深度学习 Embedding (`rag_system.py`)

**原理**：
- 使用预训练的深度学习模型（BERT、Sentence-BERT）
- 将文档转换为语义向量
- 相似语义的文本向量距离更近

**示例**：
```python
# 深度学习模型能理解语义
"汽车" 和 "车辆" 的向量非常接近
"AI Agent" 和 "智能代理" 的向量非常接近
```

**优点**：
- ✅ 强大的语义理解
- ✅ 可以处理同义词、近义词
- ✅ 多语言支持
- ✅ 生产环境标准方案

**缺点**：
- ❌ 依赖复杂（torch、transformers）
- ❌ 安装困难（依赖冲突）
- ❌ 运行速度慢
- ❌ 需要 GPU 加速（可选）

**适用场景**：
- 生产环境
- 大规模知识库
- 对语义理解要求高

## 💡 核心概念

### 什么是向量化？

将文本转换为数字向量：

```
文本: "AI Agent 是智能系统"
↓ 向量化
向量: [0.2, 0.5, 0.1, 0.0, 0.3, ...]
      ↑    ↑    ↑    ↑    ↑
     AI  Agent 智能 系统  其他
```

### 什么是余弦相似度？

计算两个向量的夹角：

```
向量A: [1, 2, 3]
向量B: [2, 4, 6]  ← 方向相同，相似度高

余弦相似度 = 1.0 (完全相同方向)

向量A: [1, 0, 0]
向量C: [0, 1, 0]  ← 方向垂直，相似度低

余弦相似度 = 0.0 (完全不相关)
```

### 为什么用余弦相似度？

- ✅ 只关注方向，不关注长度
- ✅ 适合文本相似度计算
- ✅ 值域 [0, 1]，易于理解

## 🎓 学习要点

### TF-IDF 的优势

1. **TF (词频)**：常出现的词更重要
2. **IDF (逆文档频率)**：区分度高的词更重要
3. **组合**：既考虑频率，又考虑区分度

### 示例

```
文档1: "AI Agent 是智能系统"
文档2: "AI 技术发展迅速"

查询: "Agent 系统"

TF-IDF 会：
- "Agent" 只在文档1出现 → IDF 高 → 权重大
- "AI" 在两个文档都出现 → IDF 低 → 权重小
- 因此文档1相似度更高 ✅
```
